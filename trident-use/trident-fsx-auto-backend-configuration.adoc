= Configura la configurazione backend automatica per AWS FSx per NetApp ONTAP
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Trident supporta la configurazione automatica del backend per AWS FSx for NetApp ONTAP (FSxN).

Quando si crea un StorageClass che include i parametri FSxN richiesti, Trident crea automaticamente il backend corrispondente e un VolumeSnapshotClass. Trident crea il VolumeSnapshotClass solo una volta per una determinata configurazione del driver e lo riutilizza per i successivi StorageClasses.



== Scopri come funziona la configurazione automatica del backend

Trident deriva la configurazione del backend dalla definizione di StorageClass.

Applica la StorageClass per attivare la configurazione automatica del backend. Trident esegue le seguenti azioni:

* Convalida i parametri richiesti
* Crea automaticamente il backend
* Crea un VolumeSnapshotClass (solo una volta)
* Annota lo StorageClass con lo stato di configurazione


Dopo che la configurazione ha avuto successo, puoi creare PersistentVolumeClaims che fanno riferimento a questa StorageClass.



== Specificare i parametri richiesti StorageClass

Definire i parametri richiesti nella sezione  `parameters` del StorageClass per attivare la creazione automatica del backend.

[cols="2,1,1,4"]
|===
| Parametro | Obbligatorio | Tipo | Descrizione 


| `fsxFilesystemID` | Sì | stringa | ID del file system FSx for NetApp ONTAP. 


| `storageDriverName` | Sì | stringa | Driver di storage Trident. Ad esempio: `ontap-nas` o `ontap-san`. 


| `credentialsName` | Sì | stringa | Nome o ARN del segreto di AWS Secrets Manager che contiene le credenziali. 
|===


== Specificare parametri facoltativi

È possibile passare parametri backend opzionali tramite la StorageClass.

Tutti i parametri backend supportati per FSx for NetApp ONTAP backend possono anche essere passati direttamente tramite la sezione  `parameters` del StorageClass.

[cols="2,1,1,4"]
|===
| Parametro | Obbligatorio | Tipo | Descrizione 


| Parametri del backend | Opzionale | stringa | Tutti i parametri backend supportati da FSx per NetApp ONTAP possono essere passati direttamente tramite la StorageClass `parameters` section. Ogni parametro deve essere definito come stringa. 
|===
IMPORTANTE:

* Tutti i parametri devono essere definiti come stringhe.
* Tutti i parametri devono apparire nella sezione  `parameters` del StorageClass.


Per un elenco completo dei parametri backend supportati, vedere:

xref:trident-use/trident-fsx-storage-backend.adoc#backend-advanced-configuration-and-examples[Configurazione del backend FSx for NetApp ONTAP].



== Crea un StorageClass

Crea un StorageClass che includa i parametri FSxN richiesti.

[source, yaml]
----
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: ontap-fsx-demo
  annotations:
    description: "Demo StorageClass for FSxN"
provisioner: csi.trident.netapp.io
parameters:
  fsxFilesystemID: "fs-0abc123"
  storageDriverName: "ontap-nas"
  credentialsName: "arn:aws:secretsmanager:us-west-2:123456789012:secret:trident-creds"
allowVolumeExpansion: true
reclaimPolicy: Delete
volumeBindingMode: Immediate
----
Applica il StorageClass per attivare la creazione automatica del backend.

Trident crea automaticamente le risorse backend necessarie internamente.



== Verifica lo stato della configurazione del backend

Verificare lo stato della configurazione del backend tramite le annotazioni di StorageClass.

Eseguire il seguente comando:

[source, bash]
----
kubectl get storageclass ontap-fsx-demo -o yaml
----
Esempio output:

[source, yaml]
----
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  annotations:
    trident.netapp.io/additionalStoragePools: trident-nas-sc1-fs-0752884ae35c620a0-nfs:.*
    trident.netapp.io/configuratorMessage: Completed Trident backend configuration
    trident.netapp.io/configuratorName: tconf-nas-sc1
    trident.netapp.io/configuratorStatus: Success
    trident.netapp.io/managed: "true"
parameters:
  credentialsName: arn:aws:secretsmanager:us-west-2:test-arn
  fsxFilesystemID: fs-0752884ae35c620a0
  storageDriverName: ontap-nas
provisioner: csi.trident.netapp.io
reclaimPolicy: Delete
volumeBindingMode: Immediate
----
Annotazioni chiave:

[cols="2,4"]
|===
| Annotazione | Descrizione 


| `trident.netapp.io/configuratorStatus` | Risultato della configurazione (`Success` or `Failure`). 


| `trident.netapp.io/configuratorMessage` | Stato di configurazione dettagliato o messaggio di errore. 


| `trident.netapp.io/configuratorName` | Nome risorsa configuratore interno. 


| `trident.netapp.io/managed` | Indica che la StorageClass è gestita da Trident. 


| `trident.netapp.io/additionalStoragePools` | Pool di storage creati per questo backend. 
|===
Se la configurazione riesce (`configuratorStatus: Success`, il StorageClass è completamente operativo.

Ora puoi:

* Crea PersistentVolumeClaims usando questa StorageClass
* Montare i volumi risultanti nei carichi di lavoro dell'applicazione




== Aggiungi file system FSxN aggiuntivi

Se è necessaria ulteriore capacità di archiviazione continuando a utilizzare lo stesso StorageClass, collegare ulteriori file system FSx for NetApp ONTAP.

Modifica il StorageClass:

[source, bash]
----
kubectl edit storageclass <storageclass-name>
----
Aggiungi o aggiorna l'annotazione con ID filesystem aggiuntivi:

[source, yaml]
----
metadata:
  annotations:
    trident.netapp.io/additionalFsxNFileSystemID: '["fs-03cc1a718cddd6e248"]'
----
Esempio StorageClass con un file system aggiuntivo:

[source, yaml]
----
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: test-ontap-fsx
  annotations:
    trident.netapp.io/additionalFsxNFileSystemID: '["fs-03cc1a718cddd6e248"]'
    trident.netapp.io/additionalStoragePools: trident-test-ontap-fsx-fs-0056066a48e4b22ee91-iscsi:.*
    trident.netapp.io/configuratorMessage: Completed Trident backend configuration
    trident.netapp.io/configuratorName: tconf-test-ontap-fsx
    trident.netapp.io/configuratorStatus: Success
    trident.netapp.io/managed: "true"
parameters:
  credentialsName: arn:aws:secretsmanager:us-west-2:1866785786363:secret:trident-fsx
  fsxFilesystemID: fs-0056066a48e4b22e91
  storageDriverName: ontap-san
  svmName: testSvm
provisioner: csi.trident.netapp.io
reclaimPolicy: Delete
volumeBindingMode: Immediate
----
Questa operazione aggiunge ulteriore capacità di archiviazione continuando a utilizzare la StorageClass esistente.



== Considerazioni operative e limitazioni

L'eliminazione di un StorageClass in genere elimina il backend Trident associato.

Questo può interrompere la connettività dello storage e interrompere i carichi di lavoro in esecuzione.

Verificare il potenziale impatto prima di eliminare qualsiasi StorageClass gestito.

La configurazione automatica del backend è supportata solo per AWS FSx per NetApp ONTAP.
